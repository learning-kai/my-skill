# Organization Rules

## Rewrite Checklist
- Keep the original files recoverable.
- Prefer a small number of stable categories.
- Separate by purpose before refining by date or source.
- Rename only when it improves consistency or searchability.
- Treat delete as a confirmed, high-risk action.
- Split into subfolders whenever a category contains clearly different subtypes.
- Keep each folder focused on a single responsibility.
- Never leave a mixed folder as the final state if it contains unrelated file types or unrelated tasks.

## Useful Transformations
- Messy download folder -> date/type-based archive
- Mixed project folder -> module-based structure
- Duplicate asset sets -> canonical + archive split
- Unclear names -> descriptive, consistent names
- Broad catch-all folder -> hierarchy with explicit subfolders
- Flat mixed folder -> primary category + secondary category split

## Naming Standard
- Use short, stable, sortable names.
- Prefer `YYYY-MM-DD` for dated files.
- Use one versioning style within the same folder.
- Make draft, final, backup, and temp states explicit in the name.

## Phrases to Avoid Repeating
- 一股脑地全部归档
- 直接删掉就行
- 全部放进其他
- 过度扁平化
- 过度分层
- 能平铺就平铺
- 不用建子文件夹

## Style Notes
- Favor clarity over cleverness.
- Keep the structure easy to maintain later.
- If the structure is uncertain, surface options instead of guessing.
- If a folder grows heterogeneous, create a child folder instead of letting it become a dump site.
- A folder should not remain a mixture of several responsibilities.
